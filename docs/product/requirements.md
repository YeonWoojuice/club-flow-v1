# ClubFlow Requirements Analysis

## 1. Background

동아리 운영은 매 학기 반복되지만, 실제 운영 데이터는 여러 도구에 흩어져 관리된다.

지원자는 Google Form으로 받고, 합격자 명단은 Excel로 옮기고, 부원 공지는 카카오톡으로 하고, 회비 납부는 계좌 내역과 수기 체크로 관리하는 식이다.

이 과정에서 운영진이 바뀌면 이전 학기의 맥락이 사라지고, 지원자·부원·회비·활동 기록이 서로 연결되지 않는 문제가 발생한다.

## 2. Problem Statement

ClubFlow가 해결하려는 핵심 문제는 단순한 “회원 관리”가 아니다.

핵심 문제는 다음과 같다.

- 학기별 운영 데이터가 일관되게 남지 않음
- 지원자에서 부원으로 전환되는 과정이 수동적임
- 운영진 권한과 역할이 명확히 분리되지 않음
- 회비, 활동, 부원 정보가 서로 연결되지 않음
- 다음 운영진에게 인수인계할 수 있는 구조화된 기록이 부족함
