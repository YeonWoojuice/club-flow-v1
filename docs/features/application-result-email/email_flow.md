# 지원 결과 메일 흐름

## 목적

선택한 학기의 합격자 또는 불합격자에게 같은 형식의 안내를 안전하게 보내고, 이미 보낸 사람에게 같은
결과 메일이 다시 나가지 않게 한다.

## 운영진 화면 흐름

1. 지원자 관리에서 학기를 선택한다.
2. `합격 메일 일괄 전송` 또는 `불합격 메일 일괄 전송`을 누른다.
3. 제목·본문·카카오톡 링크를 작성하고 수신 대상과 치환 결과를 미리 본다.
4. 제외 사유를 확인한 뒤 전송한다.
5. 전송 완료, 실패, 결과 확인 필요 인원수를 확인한다.
6. 실패한 수신자만 같은 버튼에서 다시 시도한다.

모달은 배경을 어둡게 하고, 전송 중에는 닫기와 중복 제출을 막는다. 키보드 Escape 닫기, 포커스 제한과
원래 버튼으로 포커스 복귀를 지원한다.

## 템플릿 변수

| 변수 | 값 | 비어 있을 때 |
|---|---|---|
| `{{clubName}}` | 동아리 이름 | 전송 제외 |
| `{{memberName}}` | 지원자 이름 | 전송 제외 |
| `{{discordName}}` | 지원자의 선택 디스코드 이름 | 이 변수를 쓴 메일만 전송 제외 |
| `{{kakaoLink}}` | 모달에서 입력한 HTTPS 링크 | 이 변수를 쓴 메일만 전송 제외 |

지원하지 않는 변수와 잘못 닫힌 중괄호는 서버에서 거절한다. 카카오톡 링크를 입력했다면 `https` 주소만
허용한다.

## 전송 대상 규칙

- 선택한 학기와 선택한 결과(`ACCEPTED` 또는 `REJECTED`)가 모두 일치해야 한다.
- `NOT_SENT`, `FAILED`만 전송할 수 있다.
- `SENT`, `PENDING`, `UNKNOWN`은 제외한다.
- 미리보기 뒤 상태가 바뀔 수 있으므로 실제 전송 직전에 서버가 대상과 권한을 다시 확인한다.
- 회장과 승인된 부회장만 전송할 수 있다.

## 처리 순서

```text
미리보기
  -> 대상·변수 검증
  -> 전송 버튼
  -> DB에 배치와 수신자별 PENDING 저장
  -> DB 작업 종료
  -> Resend 묶음 API 호출
  -> 수신자별 SENT / FAILED / UNKNOWN 기록
  -> 지원자 목록 새로고침
```

`UNKNOWN`은 시간 초과처럼 외부 제공자가 받았는지 판단할 수 없는 상태다. 바로 다시 보내면 같은 사람이
두 통을 받을 수 있어 자동 재시도하지 않는다.

## 설정

서버의 `.env.local`에만 다음 값을 둔다. 프론트엔드의 `VITE_` 환경변수에는 비밀키를 두지 않는다.

```dotenv
MAIL_ENABLED=true
RESEND_API_KEY=re_...
MAIL_FROM=CrewCat <noreply@verified-domain.example>
```

기본값은 `MAIL_ENABLED=false`다. 이 상태에서는 미리보기는 가능하지만 실제 전송은 한국어 오류로 중단한다.

## 후속 범위

- 반송, 배달, 열람 상태를 받는 제공자 웹훅
- 예약 전송
- 처리량 증가 시 작업 큐와 백그라운드 작업자
- 운영진용 발송 이력 목록과 `UNKNOWN` 수동 확인 절차
