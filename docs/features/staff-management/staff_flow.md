# 운영진 초대와 권한 관리 흐름

## 목적

회장이 Google 이메일로 부회장·운영진을 초대하고, 초대받은 사용자가 본인 계정으로 수락한 뒤
같은 동아리를 함께 관리할 수 있게 한다.

## 권한 규칙

- 승인된 회장(`PRESIDENT/APPROVED`)만 운영진을 초대·조회·변경·해제할 수 있다.
- 초대 역할은 부회장(`VICE_PRESIDENT`)과 운영진(`STAFF`)만 허용한다.
- 회장 역할은 이 기능으로 양도하거나 해제할 수 없다.
- 회장이 아닌 사용자는 운영진 관리 API를 호출할 수 없다.
- 접근 권한을 해제해도 기존 지원자·부원 데이터는 삭제하지 않는다.

## 초대 흐름

```text
회장이 Google 이메일과 역할 입력
  → 이메일을 소문자로 정리
  → 기존 승인 운영진과 대기 초대 중복 검사
  → PENDING 초대 저장
  → 초대받은 사용자가 Google 로그인
  → 로그인으로 확인된 이메일과 초대 이메일 비교
  → 수락하면 ClubStaff APPROVED 생성 또는 기존 REVOKED 행 재승인
  → 거절하면 초대를 REJECTED로 변경
```

아직 서비스에 가입하지 않은 사람도 이메일로 초대할 수 있다. 초대 메일 자동 발송은 현재 범위에
포함하지 않으며, 초대받은 사람은 CrewCat 로그인 후 받은 초대 화면에서 확인한다.

## 중복과 동시 요청

- 같은 동아리·이메일의 대기 중 초대는 DB의 부분 고유 인덱스로 하나만 허용한다.
- 같은 초대를 다시 수락해도 승인 운영진은 한 건만 유지한다.
- 초대 수락은 초대 행과 운영진 행을 순서대로 잠가 동시에 두 번 수락해도 중복 권한이 생기지 않게 한다.
- 역할 변경과 접근 해제도 운영진 행을 잠가 재초대 수락과 결과가 뒤섞이지 않게 한다.

## 역할 변경과 접근 해제

- 회장은 부회장과 운영진 역할을 서로 변경할 수 있다.
- 회장은 부회장·운영진의 접근 권한을 `REVOKED`로 바꿀 수 있다.
- 같은 권한 해제 요청을 다시 보내도 성공으로 처리한다.
- 해제된 사용자는 다음 동아리 API 요청부터 접근이 거절된다.
- 해제된 사용자를 다시 초대해 수락하면 기존 `ClubStaff` 행을 재사용한다.

## 화면 진입

- 로그인 후 대기 중인 초대가 있으면 `/staff-invitations`로 먼저 이동한다.
- 회장은 동아리 메뉴의 운영진 관리에서 현재 운영진과 대기 초대를 관리한다.
- 비회장에게는 회장 전용 관리 기능을 표시하거나 호출하지 않는다.
